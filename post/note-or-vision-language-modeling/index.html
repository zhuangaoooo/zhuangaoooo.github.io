An Introduction to Vision-Language Modeling

[2405.17247] An Introduction to Vision-Language Modeling (arxiv.org)

Abstract

Challenges needed to be addressed.

Language is discrete, vision is in a high dimensional space in which concepts cannot always be easily discretized.

Introduction

Problems unsolved in connecting language to vision:

  1. Struggle to understand spatial relationships(Fei-fei Li is focusing on spatial intelligence) or count without complicated engineering overhead that relies on additional data annotation.
  2. Lack an understanding of attributes and ordering.
  3. Hallucinate and produce content that is neither required nor relevant.

The Families of VLMs

Four different training paradigms:

  1. Contrastive training: leverages pairs of positive and negative examples.
  2. Masking: leverages reconstruction of masked image patched given some unmasked text.
  3. Pretrained backbones: leverage open-source LLMs like Llama to learn a mapping between an image encoder and the LLM.
  4. Generative.

Many approaches rely on a mix of contrastive, masking and generative criteria.

Early work on VLMs based on transformers

visual-BERT and ViL-BERT: combine text with images tokens

The models are trained on two objectives: 1) a classical masked modelling task that aims to predict the missing part in a given input; and 2) a sentence-image prediction task that aims to predict if a caption is actually describing an image content.

Contrastive-based VLMs

  1. Markov Chain Monte Carlo (MCMC): find examples that minimize the predicted energy through an iterative process.
  2. Score Matching and Denoising Score Matching criteria: remove the normalization factor by learning only the gradient of the probability density with respect to the input data.

InfoNCE:

Instead of predicting a binary value, the InfoNCE loss leverages a distance metric, such as cosine similarity, computed in a model representation space.

CLIP

Contrastive Language-Image Pre-training (CLIP): a common contrastive method using the InfoNCE loss.

The positive pairs of examples are defined as one image and its corresponding ground truth caption while the negative examples are defined as the same image but with all the other captions contained in the mini-batch that described the other images.

Incorporate vision and language in a shared representation of an image and its caption to similar embedding vectors using a contrastive loss.

SigLIP: similar to CLIP but it uses the original NCE loss.

Llip: It proposes to condition the encoding of an image on the target caption via a cross-attention module.

VLMs with masking objectives

Masking can be viewed as a specific form of denoising autoencoder in which the noise has spatial structure.

FLAVA

Foundational Language And Vision Alignment (FLAVA)

Three core components:

  1. The Image Encoder: Vision Transformer(ViT) process images into patches for linear embedding and transformer-based representation, including a classification token ([CLSI]).
  2. The Text Encoder: tokenizes textual input using a transformer [Vaswani et al., 2017] and embeds them into vectors for contextual processing and outputting hidden state vectors alongside a classification token ([CLST]).
  3. The multimodal Encoder: fuses hidden states from both the image and text encoders, leveraging learned linear projections and cross-attention mechanisms within the transformer framework to integrate visual and textual information, highlighted by an additional multimodal classification token ([CLSM]).

Generative-based VLMs

In contrast to previous training paradigms which mostly operate on latent representations to build images or text abstractions that are then mapped between each other, the generative paradigm considers the generation of text and/or images.

CoCa

An example of learning a text generator

Employ the contrastive loss and a generative loss.

Chameleon and CM3leon

CM3leon: two-stage training process.

  1. retrieval-augmented pretraining
  2. involves supervised fine-tuning (SFT), where the model undergoes multi-task instruction tuning.

Chameleon is an extension to CM3leon.
