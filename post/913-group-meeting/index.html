Data connection between cpp programme and python programme via socket

ID detection is designed to be applied on keyframes. It's benefits are as follow: 1. reduce the amount to detect and ensure the system can run real-time. 2. The position of the ID can be optimized along with the optimization of the keyframe poses.
So I edit the Tracking.cc. When a new keyframe generated, the RGB image and the deep image will be sent for detection via socket.
The disadvantage of such method is that the ID detection is still not dived from Tracking thread. I'm going to make ID a seperate thread just like Loop CLosure and Local Mapping.

Professor suggested me to try ROS2.
